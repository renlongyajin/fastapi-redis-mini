# 下一步开发计划（更新于 2025-11-12 15:45）

## 核心目标
- 打磨面试 Demo 体验：提供一键提交/查询脚本与讲解材料，让评审在 5 分钟内看到完整链路。

## 开发步骤
1. **Demo 客户端测试先行**：为 `scripts/demo_client.py` 设计测试，模拟 `POST /tasks` + 轮询流程，确保 CLI 输出可复述。
2. **实现 Demo 客户端**：编写脚本封装提交、轮询、缓存验证，并暴露命令行参数（`--host`、`--prompt` 等），让测试转绿。
3. **面试讲稿与 Q&A**：新增 `docs/面试讲稿.md`（或类似文件），概述拓扑、瓶颈与扩展；通过测试/校验确认文档存在并含关键章节。
4. **自动化脚本/Makefile**：补充 `Makefile`（或 shell 脚本）封装 `lint/test/docker compose up`，并写测试确保核心 target 存在。

## 验收标准
- `scripts/demo_client.py` 具备测试覆盖，可真实调用 API/Worker（或以 Mock 运行）。
- 文档中提供 Demo 步骤、常见 Q&A、性能口径，面试时可直接引用。
- `Makefile` 一键执行测试/构建/compose，降低手工操作风险。
