# 下一步开发计划（更新于 2025-11-12 15:10）

## 核心目标
- 完成容器化与部署脚本，让项目可 `docker compose up` 直接运行，并在文档中提供操作指引。

## 开发步骤
1. **环境配置测试先行**：为 `infra.settings` 新增单测，验证不同环境变量组合（Redis URL、队列名称）在容器化场景下的解析行为。
2. **编写 Dockerfile（API/Worker）**：先写测试（如 `pytest` 里校验生成的 Compose 模式或 build args），再实现 Dockerfile，确保镜像体积和依赖满足要求。
3. **实现 docker-compose.yml**：覆盖 API、Worker、Redis 与网络定义，同时在测试中验证 compose 文件的关键字段（端口、依赖关系）。
4. **提供运行脚本与文档**：更新 README/docs，写出一键启动、日志查看、常见问题；必要时加入 `Makefile` 或脚本，并以测试/静态校验保证存在性。

## 验收标准
- 新增的配置与 Compose 相关测试全部通过。
- `Dockerfile.api`、`Dockerfile.worker`、`docker-compose.yml` 能在本地直接构建与启动。
- 文档描述包含依赖安装、构建、启动、排错与关闭流程，便于面试演示。
